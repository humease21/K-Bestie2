

# 모바일 반응형 최적화 프롬프트 — Antigravity 전달용

---

## 목표

`ai.k-bestie.com` 랜딩 페이지를 모바일(375px 기준)에서 완벽하게 작동하도록 반응형 최적화를 수행합니다. 태블릿(768px)과 데스크톱(1280px+)에서의 기존 레이아웃은 유지하면서, 모바일에서의 가독성·터치 편의성·성능을 개선합니다.

---

## 공통 원칙

### 브레이크포인트

모바일은 max-width 767px, 태블릿은 768px~1023px, 데스크톱은 1024px 이상으로 설정합니다. 모바일 퍼스트(min-width) 방식으로 작성하되, 기존 데스크톱 스타일이 이미 있으므로 max-width 미디어쿼리로 오버라이드해도 무방합니다.

### 터치 영역

모든 버튼과 클릭 가능한 요소의 최소 터치 영역은 44px × 44px을 보장합니다. 버튼 간 간격은 최소 8px 이상입니다.

### 폰트 최소 크기

본문 텍스트는 15px 미만으로 내려가지 않도록 합니다. 소제목은 최소 18px, 섹션 제목은 최소 24px을 유지합니다.

### 좌우 여백

모바일 컨테이너 좌우 패딩은 20px로 통일합니다. 내부 요소가 화면 가장자리에 붙지 않도록 합니다.

### 이미지

모바일에서는 `max-width: 100%`, `height: auto`를 적용합니다. 폰 목업 이미지는 모바일에서 width 280px 이하로 제한하여 화면을 독점하지 않도록 합니다.

## 섹션별 수정 지침

### 1. NavBar

데스크톱의 가로 메뉴(부모 고민, 서비스 특징, 리포트, 안전 정책, FAQ)를 모바일에서는 숨기고, 우측에 햄버거 아이콘(☰)을 표시합니다. 햄버거 클릭 시 전체 화면 오버레이 또는 슬라이드 메뉴가 열리며, 각 메뉴 항목은 높이 48px 이상으로 터치하기 쉽게 배치합니다. "베타 신청하기" CTA 버튼은 햄버거 메뉴 안에도 포함하되, NavBar 우측에 작은 크기로 항상 노출되도록 합니다. 스크롤 시 NavBar는 상단 고정(sticky)하고 배경에 blur 효과를 적용합니다.

### 2. Hero 섹션

데스크톱에서 좌측 텍스트 + 우측 이미지의 2컬럼 구조를, 모바일에서는 **1컬럼 세로 배치**로 변경합니다. 순서는 헤드라인 → 서브 텍스트 → CTA 버튼 → 이미지 순입니다. 헤드라인(손글씨)은 font-size 28~32px로 조정하고, 서브 텍스트는 16~17px로 설정합니다. CTA 버튼("무료 베타 신청하기 →")은 **width 100%**로 확장하여 풀 너비 버튼으로 만들고, 높이는 52px 이상, 폰트는 17px Bold로 설정합니다. "선착순 100가정 · 7월 시작 · 완전 무료" 텍스트는 CTA 버튼 바로 아래에 중앙 정렬로 배치합니다. Hero 이미지는 CTA 아래로 이동하며, width 100%로 표시합니다.

### 3. 사회적 증거 스트립 (87+ / 4.8/5.0 / 하루 5분 × 2회 / 7월 베타 오픈)

데스크톱의 가로 4열 배치를 모바일에서는 **2열 × 2행 그리드**로 변경합니다. 각 항목은 중앙 정렬하고, 숫자(87+, 4.8/5.0 등)는 20px Bold, 라벨("사전 신청 가정" 등)은 13px로 설정합니다. 항목 간 구분선은 제거하고, 배경색으로 영역을 구분합니다.

### 4. Parents' Concerns (부모 고민 카드 4개)

데스크톱의 2열 × 2행 카드 그리드를 모바일에서는 **1열 세로 스택**으로 변경합니다. 각 카드는 이미지(높이 200px, object-fit: cover, border-radius 12px) → 소제목(18px Bold) → 설명(15px) 순으로 배치합니다. 카드 간 간격은 24px입니다. 4개 카드가 길어지므로, 3번째·4번째 카드는 접힌 상태로 두고 "더 보기" 버튼을 추가하는 방식도 고려할 수 있지만, 우선은 전체 노출로 구현합니다.

### 5. Why Not Enough (기존 방법 한계 3개)

데스크톱의 가로 3열 배치를 모바일에서는 **1열 세로 스택**으로 변경합니다. 각 항목은 소제목(18px Bold) → 설명(15px) 순으로, 항목 간 구분선(1px solid #E5E7EB) 또는 24px 간격으로 분리합니다.

### 6. Our Solution (3-Step + 폰 목업)

데스크톱의 좌측 텍스트 + 우측 폰 목업 2컬럼 구조를, 모바일에서는 **1컬럼 세로 배치**로 변경합니다. 순서는 섹션 제목 → 섹션 설명 → **폰 목업 이미지**(중앙 정렬, width 240px) → Step 1·2·3 텍스트 순입니다. 폰 목업을 텍스트보다 먼저 보여주는 이유는, 모바일에서 텍스트만 길게 나열되면 이탈률이 올라가기 때문입니다. 시각적 앵커를 먼저 제공한 후 텍스트로 설명하는 구조가 효과적입니다. 각 Step은 아이콘(32px) + 소제목(18px Bold) + 설명(15px)으로, 항목 간 간격 20px입니다.

### 7. Child Experience (아이 경험)

데스크톱 레이아웃을 모바일에서는 **1컬럼**으로 변경합니다. 아이 홈 화면 이미지를 상단에 중앙 배치(width 240px)하고, 그 아래에 "나만의 AI 친구 케이" + "매일 기다려지는 미션" 텍스트를 세로로 나열합니다.

### 8. Report Preview (리포트 미리보기 + 탭)

데스크톱의 좌측 탭 + 우측 폰 목업 구조를, 모바일에서는 **1컬럼 세로 배치**로 변경합니다. 순서는 섹션 제목 → **탭 버튼 3개를 가로 나열**(일간 | 주간 | 월간, 각 탭은 균등 분할 flex: 1, 높이 44px, 선택된 탭은 Deep Teal 배경 + 흰색 텍스트) → 탭 내용 텍스트 → 폰 목업 이미지(중앙, width 260px) 순입니다. 탭은 가로 스크롤 없이 3개가 한 줄에 들어가야 합니다.

### 9. Data Safety (안전 정책 카드 3개)

데스크톱의 가로 3열 카드를 모바일에서는 **1열 세로 스택**으로 변경합니다. 각 카드는 아이콘(32px) + 소제목(18px Bold) + 설명(15px)으로, 카드 배경은 유지하되 좌우 패딩 16px, 카드 간 간격 16px입니다.

### 10. FAQ (아코디언)

아코디언 자체는 모바일에 적합한 패턴이므로 큰 변경은 불필요합니다. 질문 텍스트는 16px Bold, 답변 텍스트는 15px로 설정합니다. 아코디언 헤더의 터치 영역은 전체 행(높이 52px 이상)이 클릭 가능하도록 합니다. 화살표 아이콘(∧/∨)은 우측 끝에 고정하고, 텍스트와 겹치지 않도록 padding-right 40px을 줍니다.

### 11. 최종 CTA

"아이의 보이지 않는 마음, 지금 시작해 보세요" 제목은 24~28px로, "선착순 100가정에게만 주어지는 무료 베타 혜택을 놓치지 마세요" 서브 텍스트는 15~16px로 설정합니다. 신청 버튼은 Hero와 동일하게 **width 100%, 높이 52px, 17px Bold**로 풀 너비 버튼으로 만듭니다. "현재 87가정 신청 완료 | 남은 인원 13명" 텍스트는 버튼 아래에 중앙 정렬로 표시합니다.

### 12. Footer

데스크톱의 2컬럼 구조를 모바일에서는 **1컬럼 세로 스택**으로 변경합니다. 순서는 로고 + 서비스 설명 → 이메일(contact@humease.com) → 서비스 링크들(세로 나열, 각 항목 높이 44px 터치 영역) → 저작권 순입니다. 전체 중앙 정렬로 통일합니다.

## 추가 모바일 최적화

### 모바일 전용 하단 고정 CTA 바

스크롤 시 화면 하단에 고정되는 CTA 바를 추가합니다. 높이 60px, 배경 흰색 + 상단 그림자(box-shadow: 0 -2px 8px rgba(0,0,0,0.1)), 내부에 "무료 베타 신청하기" 버튼을 배치합니다. Hero 영역이 뷰포트를 벗어난 시점부터 나타나고, 최종 CTA 섹션이 뷰포트에 들어오면 사라지는 방식으로 구현합니다. 이렇게 하면 사용자가 어느 위치에서든 즉시 신청할 수 있습니다.

### 이미지 최적화

모바일에서는 데스크톱용 고해상도 이미지 대신 리사이즈된 이미지를 로드하도록 `<picture>` 태그 또는 `srcset`을 사용합니다. Parents' Concerns의 4개 카드 이미지는 모바일에서 width 600px 이하 버전을 서빙하면 데이터 사용량과 로딩 속도가 크게 개선됩니다.

### 스크롤 성능

모바일에서 스크롤 애니메이션이 있다면 `will-change: transform`과 `transform: translateZ(0)`을 적용하여 GPU 가속을 활용합니다. Intersection Observer 기반 lazy-loading은 뷰포트 진입 200px 전에 트리거하여 스크롤 시 이미지가 미리 로드되도록 합니다.
